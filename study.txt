포인터는 타입맞추기 싸움이다 
&은 *이 하나 붙는 걸 꼭 기억 
*p 는 int형 근데 이게 별이 떼지면 p 이건 int *형임


 -이중포인터
 a = 집
p = 집 주소가 적힌 종이
pp = 주소가 적힌 종이 위치
주소 종이를 바꾸고 싶다
종이의 위치(pp)가 필요

| 표현     | 의미    |
| ------ | ----- |
| `p`    | a의 주소 |
| `*p`   | a     |
| `pp`   | p의 주소 |
| `*pp`  | p     |
| `**pp` | a     |


이중포인터를 쓰는 이유는
#include <stdio.h>

void change(int x)
{
    x = 20;
}

int main()
{
    int x = 10;

    change(x);

    printf("%d", x);
}

이런 식이면 x는 여전히 10임 
왜냐면 void로 x를 받은 건 값을 복사하는 거라서
그래서 그 값을 바꾸려면 주소를 받아야함

TIP 포인터는 상자로 생각하라
상자의 주소를 내놔 
void change(int **pp)// pp->p->a
{
    int b = 20;
    *pp = &b; //우린 p가 가르키고 있는 주소를 변경하고 싶은 거니까 *pp (p에접근하기위함) 왜 &을 썼냐 pp는 int **, *p는 int *이니 타입명을 맞추기위해 int *인 &b 
}

main

int a = 10;
int *p = &a;

change(&p);
일케 해야함

별과 타입을 생각하라 밑을보셈

//이건 삼중포인터 
#include <stdio.h>

void changeDoublePointer(int ***ppp) { ppp ──► pp ──► p ──► a
    static int b = 20;               int***   int**  int*   int 
    static int *newp = &b;   // 새로운 포인터
    static int **newpp = &newp; // 새로운 이중포인터

    *ppp = newpp;   // pp 자체를 newpp로 변경 //*ppp의 의미는 pp겠지 타입은 int**그래서 저 위의 타입을 맞춰준것 newpp도 별을 안 붙혔으니 타입 int**
    즉! *ppp = &newp; 와 같은 말이다 
}

int main() {
    int a = 10;

    int *p = &a;
    int **pp = &p; //참고로 선언을 할 땐 int **형인거임 선언하고 나서 **pp 쓰면 그냥 int형 pp이렇게 써야 **int형임 pp의 타입 = int **이라고 선언순간에 컴파일러가 처리함 

    printf("before: %d\n", **pp);

    changeDoublePointer(&pp);  // 삼중포인터 전달

    printf("after: %d\n", **pp);

    return 0;
}

pp p의 주소 즉 pp = &p
*pp pp가 가리키는 값 즉 *pp = p , 출력은 a의 주소
**pp는   pp → p → a 즉 a의 값
