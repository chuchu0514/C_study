1. 시간 복잡도: "최악의 상황에서 반복문이 몇 번 도나?"시간 복잡도는 실제 시간이 아니라 연산 횟수입니다. 
코드에서 가장 많이 실행되는 부분(보통 반복문 안쪽)만 찾으세요.상수 시간 $O(1)$: 반복문이 없거나, 
데이터 개수와 상관없이 정해진 횟수만 돌 때.선형 시간 $O(n)$: 데이터 $n$개만큼 정직하게 반복문이 돌 때. 
(방금 본 sum 함수)이차 시간 $O(n^2)$: 중첩 반복문(for문 안에 for문)이 돌 때.로그 시간 $O(\log n)$: 
반복할 때마다 처리해야 할 양이 절반씩 팍팍 줄어들 때 (예: 이진 탐색).팁: $n$이 1억(10^8)번 정도 돌면 보통 1초 정도 걸린다고 생각하고 문제를 풀면 감이 빨리 옵니다.
2. 공간 복잡도: "변수와 스택의 개수를 세라"공간 복잡도는 프로그램이 실행되면서 추가로 사용하는 메모리 양입니다.반복문: 변수 몇 개(i, sum)만 새로 만드니까 데이터가 
100만 개여도 공간은 $O(1)$입니다.재귀함수: 아까 본 코드처럼 함수가 호출될 때마다 스택 프레임이 쌓입니다. $n$번 호출되면 메모리에 $n$개의 층이 쌓이므로 $O(n)$이 됩니다.
배열 생성: malloc이나 new로 $n$ 크기의 배열을 만들면 당연히 $O(n)$입니다

void push(element item) {
    // 꽉 찼으면 doubling
    if (top >= capacity - 1) {
        capacity *= 2;
        stack = (element*)realloc(stack, capacity * sizeof(element));
    }
    stack[++top] = item;
}

ex)
push 1번 → 여유있음 (capacity=1, top=-1→0)
push 2번 → capacity 1→2 doubling → 1개 copy
push 3번 → 여유있음
push 4번 → capacity 2→4 doubling → 2개 copy
push 5번 → 여유있음
push 6번 → 여유있음
push 7번 → 여유있음
push 8번 → capacity 4→8 doubling → 4개 copy

push가 N번일 때 doubling 횟수 = log₂N
push 17이면 doubling횟수 5 소숫점 있으면 무조건 올림 
doubling이 k번이면 total copy 횟수는 {2(k승) - 1} ->copy에 대한 시간복잡도는 그래서 O(2(k승)
근데 push가 n번일 때 dougling은 log₂N 번이니까 k = log₂N  즉 O(N)

!! 비어있는 조건을 알면 쉽다

Linear queue: front=0, rear=-1
rear가 마지막 데이터 위치를 가리켜.
처음엔 데이터가 없으니까 -1 (없음을 표현)
add하면 queue[++rear] = num  ← rear 먼저 증가 후 저장
비어있는 조건: front > rear

Circular queue: front=0, rear=0
front랑 rear 둘 다 빈 슬롯을 가리켜.
처음엔 같은 위치(0)에서 시작
add하면 crear = (crear+1) % size 후 저장
비어있는 조건: cfront == crear
