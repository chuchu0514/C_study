void push(element item) {
    // 꽉 찼으면 doubling
    if (top >= capacity - 1) {
        capacity *= 2;
        stack = (element*)realloc(stack, capacity * sizeof(element));
    }
    stack[++top] = item;
}

ex)
push 1번 → capacity 1→2 doubling → 1개 copy
push 2번 → capacity 2→4 doubling → 2개 copy
push 3번 → 여유있음
push 4번 → capacity 4→8 doubling → 4개 copy
push 5번 → 여유있음
push 6번 → 여유있음
push 7번 → 여유있음
push 8번 → 여유있음

push가 N번일 때 doubling 횟수 = log₂N
push 17이면 doubling횟수 5 소숫점 있으면 무조건 올림 
doubling이 k번이면 total copy 횟수는 {2(k승) - 1} ->copy에 대한 시간복잡도는 그래서 O(2(k승)
근데 push가 n번일 때 dougling은 log₂N 번이니까 k = log₂N  즉 O(N)

!! 비어있는 조건을 알면 쉽다

Linear queue: front=0, rear=-1
rear가 마지막 데이터 위치를 가리켜.
처음엔 데이터가 없으니까 -1 (없음을 표현)
add하면 queue[++rear] = num  ← rear 먼저 증가 후 저장
비어있는 조건: front > rear

Circular queue: front=0, rear=0
front랑 rear 둘 다 빈 슬롯을 가리켜.
처음엔 같은 위치(0)에서 시작
add하면 crear = (crear+1) % size 후 저장
비어있는 조건: cfront == crear
